# Frontend Mentor - Shortly URL shortening API Challenge React
Deployment link = https://url-shortening-project-react.netlify.app/

This project challenged me to combine responsive layout design with real API integration and browser storage, making it one of the most complete frontend applications I’ve built so far. I started by building the project with HTML, CSS, and JavaScript using a mobile-first approach. I focused on structuring the layout clearly and matching the design as closely as possible, especially with spacing, background images, and responsive behavior. After getting the UI working, I added functionality like form validation, shortening links using the Bitly API, and saving results in local storage so they persist after refresh. Once everything was working, I refactored the entire project into React, breaking the UI into reusable components and organizing the code into separate folders for components, services, and utilities.

One of the biggest challenges was working with the Bitly API. Understanding how to send requests with the correct headers and handle responses took some trial and error. I also had to be careful about API limits, so I used a mock mode during development to avoid using too many requests. Another challenge was transitioning from vanilla JavaScript to React, especially managing state and passing data between components. I also ran into issues during deployment, such as images not loading due to incorrect paths and case sensitivity differences. Fixing these helped me better understand how projects behave differently in local vs production environments.

If I were to improve this project further, I would move the API request to a backend service to keep the API key secure. I would also continue improving the structure by adding custom hooks and possibly more features like deleting links or showing analytics. Overall, this project helped me build confidence with React, APIs, and creating a complete, responsive web application.
